# Security Policy

> ⚠️ **This project is no longer actively maintained.**
> It was developed as a university assignment and has officially concluded. No further updates, patches, or security fixes will be issued for any version.

## Supported Versions

No versions of **Jadwalku** are currently receiving security updates.

| Version | Supported |
| ------- | --------- |
| 1.x   | :x:       |

---

## Reporting a Vulnerability

Even though this project is no longer maintained, we appreciate responsible disclosure. If you discover a security vulnerability in Jadwalku, please **do not** open a public GitHub issue.

### How to Report

You may submit a private report through one of the following:

- **Email:** `ramdan@nyanpixel.my.id`
- **GitHub Private Advisory:** Use the [Security Advisories](https://github.com/kelompok/jadwalku/security/advisories/new) feature in this repository

### What to Include

To help us understand the report, please provide:

- A brief description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- The affected version(s)
- Supporting evidence (screenshots, logs, or proof of concept if available)

### What to Expect

Because this project is no longer active, **no patches or fixes will be released**. However, we will still acknowledge your report and respond within a reasonable timeframe to confirm receipt.

| Status | Response |
| --- | --- |
| Vulnerability **accepted** | We will acknowledge it, but no fix will be issued |
| Vulnerability **declined** | You will receive an explanation |
| **Duplicate** report | You will be referred to the existing report |

### Scope

Vulnerabilities that would have been relevant to this project include:

- User data leaks (Firebase Auth / Firestore)
- Remote Code Execution
- Authentication or authorization bypass
- Misuse of notifications or AlarmManager
- Malicious data injection into Firestore

The following are **out of scope:**

- UI/display bugs unrelated to security
- Feature requests
- Performance issues

## Disclosure Policy

We follow **Responsible Disclosure** practices. Please do not publicly disclose any vulnerability before receiving a response from us.
